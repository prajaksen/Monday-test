Local Setup

1. Start Minikube

minikube start

2. Add the Langfuse Helm repository

helm repo add langfuse https://langfuse.github.io/langfuse-k8s
helm repo update

3. Create a local values file

Create a file named values-local.yaml locally and populate it with the required secrets. Do not commit this file.

4. Deploy Langfuse

helm install langfuse langfuse/langfuse \
  --namespace langfuse \
  --create-namespace \
  -f values-local.yaml

5. Verify pods

kubectl get pods -n langfuse

6. Access the UI

kubectl port-forward svc/langfuse-web 3000:3000

Then open http://localhost:3000 in a browser.

7. Run the webhook locally

python webhook.py

The webhook listens on port 5000 by default.

8. Verify the webhook is reachable

curl http://127.0.0.1:5000/healthz

Expected response:

{"status":"ok"}

If you see ERR_CONNECTION_REFUSED, the service is not running yet or the port-forward command is not active.

- Start the webhook locally with: python webhook.py
- Or expose the Kubernetes service locally with:

kubectl port-forward svc/langfuse-webhook 5000:80 -n langfuse

Then test again with:

curl http://127.0.0.1:5000/healthz

For Monday automation, use the webhook endpoint:

POST http://127.0.0.1:5000/monday-webhook
